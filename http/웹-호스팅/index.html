웹 호스팅

Sun, Sep 15, 2019

18 장 웹 호스팅

1. 호스팅이란

웹 호스팅이란 웹 컨텐츠를 포함하는 리소스를 저장, 중개, 관리하는 일을 말한다. 웹 호스팅에 필요한 하드웨어를 직접 관리하기 어려운 경우 호스팅 서비스나 호스팅 업체를 이용한다. 웹 호스팅 서버와 http 어플리케이션이 어떻게 상호작용 하는지를 알 필요가 있다. 전용 호스팅 서비스를 사용한다면, http 어플리케이션은 물리적인 서버를 단독으로 배정받는다. 브라우져에서는 호스팅 서비스의 ip를 가리키는 주소에 http 요청을 하게 된다.

2. 가상 호스팅

현대에는 많은 사람들이 웹 공간을 가지고 싶어하기 때문에, 호스팅 서비스 제공자는 비용하락을 위해서 물리적으로 하나의 서버에 여러개의 http 어플리케이션을 호스팅한다. 트래픽이 올라가서 전용호스팅이 필요해 지기 전까지 가상 호스팅 서비스를 이용하게 된다. 서비스 제공자는 하나의 컴퓨터에 많은 어플리케이션을 서비스하는 것은 아니며, 서버팜을 이용해서 부하를 관리한다.
가상 호스팅을 적용하기 위해서는 요청에서 어느 사이트에 접근하려고 하는지에 대한 정보가 필요하다. 이 정보를 활용해서 http 어플리케이션을 특정하여 요청을 보내게 된다. 이는 다음과 같은 네가지 방법으로 실현될 수 있다. - url 경로를 이용한 가상 호스팅 - 포트번호를 이용한 가상 호스팅 - ip 주소를 이용한 가상 호스팅 - host 헤더를 이용한 가상 호스팅

3. 안정적인 서버관리

웹 사이트에는 서버 다운, 트래픽 폭증, 네트워크 장애와 같은 문제들이 생긴다. 이러한 문제들에 대응하기 위해서 미러링 된 서버팜을 이용하거나, 컨텐츠 분산 네트워크를 이용하는 등의 기술적인 방법들을 사용한다.

4. 웹사이트의 로딩 속도를 빠르게 하기

자원을 분산시켜서 사용자에게 제공하는 것은 그 자체로 사용자와 자원 사이의 거리를 가깝게 만들기 때문에 로딩 속도를 향상 시킨다. 이외에도 컨텐츠를 압축해서 전송한 이후 클라이언트에서 압축해제하도록 해서 자원의 전송속도를 향상 시킬 수 있다.
