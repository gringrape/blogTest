토비의 스프링 스터디 1주차 내용 정리

Tue, Aug 6, 2019

7월 20일 진행한 1차 모임 전후에 대한 기록.

독서 전략

높은 추상화 단계에서 내용 파악 -> 추상화 단계를 낮추어서 세부 내용 파악

아래는 구체적인 행동 순서.

  • 목차를 통한 내용 파악
  • 주요 용어 골라내기, 의미파악
  • 1차 내용 정리
  • 세부적인 의미파악
  • 반복적인 스키밍을 통한 의미정착
  • 2차 세부 내용 정리

세부 내용 정리

template method pattern 을 통한 기능의 확장

  • 변하지 않는 기능을 superclass 에 정의 (template method)
  • 자주 변하는 기능(확장)을 subclass 에서 구현
  • hook method 와 abstract method
    • hook method : 기본적인 내용을 구현하거나 비워놓았다가 선택적 으로 override 해주는 것.
    • abstract method : 반드시 subclass에서 구현되어야함.

strategy pattern 을 통한 기능의 확장

  • 하나의 클래스를 context + strategy로 보고 클래스를 분리.
  • context 에서 strategy의 interface를 의존.
  • 사용 책임과 생성 책임의 분리
    • strategy 객체를 생성할 책임을 factory 라는 객체로 분리.
    • 런타임에서 context에 동적으로 strategy를 만들어 준다.

factory 와 제어의 역전

  • 객체가 스스로 어떤 다른 객체에게 의존할지 선택하지않고 의존하는 객체를 생성하지도 않는다.
  • 객체는 모든 제어 권한을 다른 대상에게 위임한다.
  • 스프링에서 스프링 컨테이너는 제어권한을 위임받아 설정정보 를 참조하여 객체를 생성하고 주입한다.
